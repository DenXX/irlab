Reasons for failure

The Reliable Information Access Workshop search failure categorization:
  1. General technical failure (stemming, tokenization, etc.)
  2. All systems emphasize one aspect, missing another required term
  3. All systems emphasize one aspect, missing another required aspect
  4. Some systems emphasize one aspect, some another, need both
  5. All systems emphasize one irrelevant aspect, missing point of topic
  6. Need outside expansion of general term
  7. Need QA query analysis and relationships
  8. Systems missed difficult aspect that would need human help (what is new method ...)
  9. Need proximity relationship between two aspects ("What countries are experiencing increase ...)

Why interesting?

Understanding the reason of failure can:
  • lead to more robust retrieval
  • be used to assist users in difficult search tasks

How users react on search tips?

It was previously shown, that search tips can be effective in reducing the search time and they have educational aspect.
  • Task: Imagine you live in Centreville, MD. What library within 15 miles of Centreville has the book The Person and the Situation and how far away is it (in miles)?
    • Tip: You can search for books in a local library with Google and see if that book is in a local library by clicking the "Find in a library" link.
